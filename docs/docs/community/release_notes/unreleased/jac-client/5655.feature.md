- **Feat: `jacLogin`/`jacSignup` accept identity/credential dicts**: Both functions now accept either the existing `(username, password)` string pair (backward-compatible) or the richer `(identity_dict, credential_dict)` / `(identities_list, credential_dict)` form — e.g. `jacLogin({"type": "username", "value": "alice"}, {"type": "password", "password": "s3cret"})`. Internally both paths now send the new `{identity, credential}` wire payload that jaclang and jac-scale servers speak.
