- **Fix: `jacLogin` / `jacSignup` now send the identity/credential shape jac-scale expects**: The client was POSTing a legacy `{username, password}` body that the server no longer accepts. Both functions now send `{identity: {type, value}, credential: {type, password}}` (plural `identities: [...]` for signup). Two new types, `Identity` and `Credential`, are exported from `@jac/runtime` for explicit calls like `jacLogin(Identity.email("a@b.com"), Credential.password("pw"))`. Existing 2-arg calls such as `jacLogin("alice", "pw")` keep working unchanged. Registration now also accepts multiple identities: `jacSignup([Identity.username("alice"), Identity.email("a@b.com")], "pw")`.
