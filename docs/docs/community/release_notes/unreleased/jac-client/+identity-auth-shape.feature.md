- **Fix: `jacLogin` / `jacSignup` now speak jac-scale's identity/credential shape**: The client was still POSTing the legacy `{username, password}` body, which has been rejected by jac-scale since #5424. Both functions now send the `{identity: {type, value}, credential: {type, password}}` (or `identities: [...]` for signup) shape expected by the server. New `Identity` and `Credential` types are exported from `@jac/runtime` for explicit construction: `jacLogin(Identity.email("a@b.com"), Credential.password("pw"))`. Existing 2-arg calls like `jacLogin("alice", "pw")` keep working — strings are auto-wrapped (presence of `@` means email, else username; password strings become `Credential.password(...)`). Multi-identity register is now supported via `jacSignup([Identity.username("alice"), Identity.email("a@b.com")], "pw")`.
