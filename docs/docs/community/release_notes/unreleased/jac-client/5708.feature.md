- **Feat: `jacSignup` accepts optional `profile` and auto-logs-in after register**: `jacSignup(identities, credential, profile?)` now takes an optional third arg, a `profile` dict forwarded to `POST /user/register` for fields like firstname, lastname, address, postcode. The profile is omitted from the request body when null. Successful signup also automatically calls `jacLogin` so the caller ends up authenticated with a token in `localStorage["jac_token"]` (previously the client returned `"No token received"` because `/user/register` does not issue tokens). To read the profile, call `GET /user/me` directly with the Bearer token.
