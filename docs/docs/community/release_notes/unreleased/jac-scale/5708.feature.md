- **Add: Optional `profile` on register and new `GET /user/me` endpoint**: `POST /user/register` now accepts an optional `profile: dict` of arbitrary developer-defined fields (firstname, lastname, address, postcode, etc.) alongside `identities` and `credential`. Values must be strings, numbers, or booleans, with bounded limits (≤20 keys, ≤64 chars per key, ≤1024 chars per value, ≤8192 bytes total) and keys restricted to `[a-zA-Z][a-zA-Z0-9_]*` to prevent MongoDB operator/dot-path injection and JS prototype pollution. Profile is stored as a `profile` sub-document on the user record (Mongo and SQLite backends). New `GET /user/me` returns the authenticated user's `{user_id, role, status, identities, profile, created_at, updated_at}` (Bearer-auth, credentials and normalized identity values stripped).
