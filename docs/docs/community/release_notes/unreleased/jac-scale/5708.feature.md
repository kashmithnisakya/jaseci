- **Add: Optional `profile` on register, new `GET /user/me`, and SSO profile population**: `POST /user/register` accepts an optional `profile: dict` of arbitrary developer-defined fields (firstname, lastname, address, postcode, etc.). Values must be strings, numbers, or booleans, with bounded limits (20 keys, 64 chars per key, 1024 chars per value, 8192 bytes total) and keys restricted to `[a-zA-Z][a-zA-Z0-9_]*` to prevent MongoDB operator/dot-path injection and JS prototype pollution. Profile is stored as a `profile` sub-document on the user record (Mongo and SQLite backends). The new `GET /user/me` returns the authenticated user's `{user_id, role, status, identities, profile, created_at, updated_at}` (Bearer-auth; credentials, password hashes, normalized identity values, and SSO `external_id` are stripped). SSO providers (Google, GitHub, Apple) now populate `profile.sso.<platform>` from the OAuth response (`display_name`, `first_name`, `last_name`, `picture`) using the URL platform as the single source of truth; `sso_callback` refreshes them on every login so they stay current without touching developer-set fields outside the `sso` namespace. `SSOUserInfo` gains those four optional fields and exposes a `profile_fields()` method; the unused `SSOUserInfo.platform` field and `SSOProvider.get_platform_name()` abstract method were removed. SSO identities in the `/user/me` response expose `provider` instead of an empty `value`. Example user document: `{user_id, status, role, identities: [{type: 'email'|'sso', ...}], credentials, root_id, profile: {firstname, lastname, sso: {google: {display_name, first_name, last_name, picture}, github: {display_name, picture}}}, created_at, updated_at}`. Example `/user/me` response: `{ok: true, data: {user_id, role, status, identities: [{type:'email', value, verified, is_recovery}, {type:'sso', provider, verified, is_recovery}], profile: {firstname, lastname, sso: {google: {...}}}, created_at, updated_at}}`. See [docs/reference/plugins/jac-scale.md](../../../reference/plugins/jac-scale.md) for full request/response examples.
