**Scheduler hardening**:

- Dynamic jobs now run as the creating user (stable `user_id`); static jobs continue to run as `__system__`.
- System-role accounts are no longer loginable over HTTP.
- `/jobs` endpoints enforce per-job ownership on `GET`/`PUT`/`DELETE`.
- `/jobs` listing supports `limit` / `offset` / `trigger` / `created_by` pagination.
- Jobs are persisted before scheduling; MongoDB is authoritative when configured.
- APScheduler mutations serialized via an `RLock`.
- Cron expressions validated via `CronTrigger.from_crontab`.
- Thread-pool size, misfire grace, and shutdown drain timeout configurable under `[plugins.scale.scheduler]`.
- Scheduled jobs record `last_run_at`, `last_status`, `last_error`, and `run_count`.
- Graceful shutdown drains in-flight tasks on `SIGTERM`.
