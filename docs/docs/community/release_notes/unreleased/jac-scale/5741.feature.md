- **Add: User identity management, password reset, and pluggable emailer**: Four new endpoints under `/user/*`, plus an `Emailer` abstraction so any SMTP / SendGrid / Mailgun backend can be plugged in via `jac.toml`. Reset and verification tokens are SHA256-hashed at rest, single-use, and TTL-bounded; persisted in MongoDB when configured (with a TTL index) or in-memory otherwise.

  **Configuration (`jac.toml`):**

  ```toml
  [plugins.scale.emailer]
  provider = "smtp"                      # 'smtp', a registered name, or 'pkg.module:ClassName'
  from_address = "noreply@example.com"

  [plugins.scale.emailer.smtp]
  host = "smtp.example.com"
  port = 587
  username = "apikey"
  use_tls = true
  # password = "..."  (or set EMAILER_SMTP_PASSWORD env var)

  [plugins.scale.auth]
  reset_token_ttl_seconds = 1800
  verify_token_ttl_seconds = 86400
  reset_url_template = "https://app.example.com/reset?token={token}"
  verify_url_template = "https://app.example.com/verify?token={token}"
  ```

  **`POST /user/add-identity`** - attach a new identity to the authenticated user. Email identities trigger a verification mail and start unverified. Requires a Bearer token.

  Request:

  ```json
  {
    "identity": {"type": "email", "value": "alice@example.com"},
    "is_recovery": false
  }
  ```

  Response (email, `202 Accepted`):

  ```json
  {
    "ok": true,
    "data": {"status": "pending_verification", "verified": false, "email_sent": true},
    "error": null,
    "meta": {"extra": {"http_status": 202}}
  }
  ```

  Response (username, `200 OK`):

  ```json
  {
    "ok": true,
    "data": {"status": "added", "verified": false},
    "error": null,
    "meta": {"extra": {"http_status": 200}}
  }
  ```

  Errors: `401 UNAUTHORIZED`, `409 IDENTITY_TAKEN`, `503 EMAIL_DISABLED` (email type when no emailer is configured), `404 NOT_FOUND`.

  **`POST /user/verify-identity`** - consume a verification token from the email and flip the matching identity to `verified=true`. No auth required (the token is the credential).

  Request:

  ```json
  {"token": "Hk7sQpL2x9aRnZ8Dm4e3vT7g..."}
  ```

  Response (`200 OK`):

  ```json
  {
    "ok": true,
    "data": {"status": "verified", "identity": "alice@example.com"},
    "error": null,
    "meta": {"extra": {"http_status": 200}}
  }
  ```

  Errors: `400 INVALID_TOKEN` (expired, already-consumed, or unknown).

  **`POST /user/forgot-password`** - issue a one-time reset token to the user's verified recovery email. **Always returns `200`** to avoid leaking whether the account exists.

  Request:

  ```json
  {"identity": {"type": "email", "value": "alice@example.com"}}
  ```

  Response (`200 OK`, regardless of whether the account exists):

  ```json
  {
    "ok": true,
    "data": {
      "status": "ok",
      "message": "If that account exists, a reset link has been sent."
    },
    "error": null,
    "meta": {"extra": {"http_status": 200}}
  }
  ```

  **`POST /user/reset-password`** - consume a reset token and set a new password. Other outstanding reset tokens for the same user are revoked on success.

  Request:

  ```json
  {"token": "Hk7sQpL2x9aRnZ8Dm4e3vT7g...", "new_password": "newSecret123"}
  ```

  Response (`200 OK`):

  ```json
  {
    "ok": true,
    "data": {"status": "password_reset"},
    "error": null,
    "meta": {"extra": {"http_status": 200}}
  }
  ```

  Errors: `400 INVALID_TOKEN`.

  **Custom emailer (Python or Jac):** subclass `jac_scale.abstractions.emailer.Emailer`, then point `provider` at it:

  ```toml
  [plugins.scale.emailer]
  provider = "myapp.email:SendGridEmailer"
  ```

  The factory imports it via `importlib` at startup and validates that the class subclasses `Emailer`. No jac-scale code changes required.
