- **Fix: type narrowing now reaches comprehensions inside `if x is not None` (#6731)**: A comprehension in the body of an `if x is not None { ... }` guard no longer raises a spurious `E1001` (e.g. `Cannot assign list[NoneType] to list[Box]`). Inside the guard, `x` is correctly treated as non-`None`, so `[x for _ in range(3)]` type-checks as expected, for list, set, dict, and generator comprehensions. Genuine errors are still reported: dereferencing `x` in a branch where it really is `None` continues to error.
