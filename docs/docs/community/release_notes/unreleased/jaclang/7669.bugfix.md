- **jac-scale in-pod client build no longer runs the strict type-check gate (#7667)**: the gateway pod rebuilt the client with a bare `jac build "<entry>"`, which triggered the whole-project type-check gate and failed on any `error[Exxxx]` (including the Any-typed noise `.cl.jac` files routinely emit), so type-error apps deployed green but served a JSON 404 at `/`. The pod build now uses `jac build "<entry>" --client web`, matching the host build and bypassing the gate, and a genuine client-build failure is no longer swallowed by `|| echo` so it surfaces the deploy as failed instead of silently shipping headless.
