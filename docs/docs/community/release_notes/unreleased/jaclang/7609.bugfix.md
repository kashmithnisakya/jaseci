Fixed `jac run` and `jac start` hanging indefinitely when `MONGODB_URI` pointed at an unreachable MongoDB. Mongo clients are now built with a bounded `serverSelectionTimeoutMS` (new `mongodb_timeout_ms` config key, default 5000, overridable via `MONGODB_TIMEOUT_MS`), and both the persistence and identity layers probe once before committing to Mongo instead of discovering the failure one blocking round-trip at a time.
