- **Fix: `jac start` auth endpoints accept both legacy and identity/credential shapes**: The built-in `/user/register` and `/user/login` handlers now read credentials from either the legacy `{username, password}` body or the newer `{identity: {type, value}, credential: {type, password}}` body. This keeps `jac-client`'s `jacLogin` / `jacSignup` wire-compatible with `jac start` as well as `jac-scale`, so apps using either backend work without changes.
