- **Feature: admin Services monitoring view**: Adds a "Services" tab to the admin portal's Monitoring section. The list view (`ServicesPage`) shows the services in the app's namespace, and a detail view (`ServiceDetail`) drills into a single service with its pods and CPU/memory history. The `MicroserviceGateway` now inherits the workloads mixin so the gateway serves the data the Services page reads. The former flat "Workloads" and "Usage" tabs are removed; both are now covered by Services and its drill-down.
- **Fix: admin Metrics page works on gateway deployments**: On microservices (gateway) deployments the Metrics page reported "off" because the gateway never created an HTTP metrics collector, and the admin endpoint read the global Prometheus registry instead of the collector's own. The gateway now wires up a collector (fed per request by the dispatcher) and `/admin/metrics` reads that collector's registry via a new `MetricsCollector.expose()`.
