- **Scale: webhook key durability warning only fires when webhooks are in play**: the "Webhook API keys are stored in memory" warning no longer appears on every `jac start` without MongoDB. `ApiKeyManager` is now constructed lazily, and the warning surfaces at boot only for apps that actually register webhook walkers (or on first API key use). The microservice gateway's unused eager construction, which produced the same spurious warning in local microservice mode, was removed.
- **Scale: client-side `sv` imports no longer trigger local microservice mode**: service discovery now ignores `sv` imports that live in client code (`.cl.jac` modules or `cl` blocks), so a page calling a `def:pub` through `sv import` no longer flips the whole app into microservice mode. Server-side `sv` imports behave exactly as before.
