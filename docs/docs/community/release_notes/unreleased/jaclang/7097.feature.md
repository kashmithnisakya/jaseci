- **`jac bundle` produces self-sufficient pure-Python wheels**: `jac bundle` transpiles every server-codespace `.jac` source to `.py` at build time and ships a wheel (and sdist) containing only Python, so the artifact installs with plain `pip` and imports in an environment with no jac binary and no jaclang. Three mechanisms make the wheel actually run, not just install: intra-project imports are re-pointed at their installed locations by resolving each one through the compiler's own module resolver (a flat project's `import helper` becomes `from mypkg import helper`; already-package-qualified imports are untouched); projects whose transpiled code needs the jac runtime get a compiled runtime closure vendored under `<pkg>/_jac_vendor/` with a bootstrap in `__init__.py` that prefers a host jaclang and falls back to the vendored copy (declaring `jaclang` in `[dependencies]` skips vendoring for jac-hosted deployments); and the sdist's PEP 517 backend is now stdlib-only with the build's metadata baked in, so `pip install` from the sdist works in an isolated build env. Client/native codespace files (`.cl.jac`/`.na.jac`) are excluded from wheels with a notice instead of shipping as empty modules, flat-layout projects (default `jac create` scaffold) collect recursively and package correctly under a synthesized package, `top_level.txt` reflects actual wheel contents, and a `.jac` that fails to compile aborts the build with the compiler's errors instead of writing a partial artifact. The interim `--target python` is removed; the `wheel` target is pure-Python by default.
