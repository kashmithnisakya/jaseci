- **jac-scale: removed the inert `[scale.kubernetes]` pinning keys**: `jaseci_repo_url`, `jaseci_branch`, `jaseci_commit`, `install_jaseci`, and `plugin_versions` were parsed but honored by no deploy path, and the per-service `image_tag` override was documentation-only. A `jac.toml` still carrying them keeps working with the keys silently ignored. To pin the runtime pods run, use `[scale-runtime] version` (or `JAC_SCALE_VERSION`), added in #7698.
