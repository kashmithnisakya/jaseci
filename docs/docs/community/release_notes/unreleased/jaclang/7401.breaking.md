- **Breaking: Removed the `to cl:` / `to sv:` / `to na:` section-marker syntax (#7401)**: The module-level colon-section-marker form that put every following statement into a client/server/native context until the next marker has been removed with no deprecation path. Writing `to cl:` (or `to sv:` / `to na:`) is now a parse error. Migration: wrap the region in the braced block form instead (`cl { ... }`, `sv { ... }`, `na { ... }`), which parses to the same `ClientBlock`/`ServerBlock`/`NativeBlock` node and is now the canonical form; server code can also move to module top level since server is the default context. The single-statement prefix forms (`cl def:pub foo() {...}`, `sv ...`, `na ...`) and the file-extension contexts (`.cl.jac`, `.na.jac`) are unaffected. The `implicit` field on the three block AST nodes (only ever set by the removed marker) was dropped, and `to` remains valid in its unrelated iter-for use (`for x = 0 to 10 by 1`).
