- **K8s pods no longer cold-compile the app's modules on first boot (any channel)**: #7144's `.jir` seeds rode the `[dev]` overlay only, so stable-channel deploys (every production app) still compiled the whole app once per pod before serving (#7179). `pack_source` now stages the project, precompiles it with the **already-downloaded pod binary** (`.jir` is version-locked, so the driver's own jac cannot produce pod-valid artifacts), and ships `_precompiled/cpython-XY/*.jir` in the source bundle. The core loader already discovers and content-validates `_precompiled/` beside the modules, so pods need no changes - they load instead of compiling, on stable and dev alike. Fully best-effort: if the precompile can't run, the bundle ships source-only and pods compile exactly as before.
