- **Feature: `jac eject` produces a standalone, zero-jaclang app**: `jac eject` now compiles a Jac project into a project-internal `ejected/` folder split into a FastAPI `backend/` (walkers and functions compiled to Python that import a vendored `_osp.py` runtime instead of jaclang) and a `frontend/` (the `.cl.jac` UI compiled to JavaScript with a Vite setup). The vendored `_osp.py` runs walkers and an in-memory object-spatial graph (nodes, `connect`/`++>` edges, `visit`/`OPath` traversal, `report`), and the backend serves the `@jac/runtime` HTTP contract plus the built frontend, so one `uvicorn main:app` serves the whole app with only `fastapi` + `uvicorn` installed (no Jac toolchain or `jaclang`). Re-running regenerates the folder without `--force` (it carries a `.jac-ejected` marker), and the output directory is overridable via `[eject].output` in jac.toml.
