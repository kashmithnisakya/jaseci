- **Fix: jac-desktop routes walker/function invokes through inprocess dispatch**: `window.__jac_invoke` requests previously had no path to the in-process walker/function runtime. All requests now go through one dispatch entry point: `{"kind": "walker" | "function"}` payloads route to `inprocess_dispatch.dispatch_sync`, everything else to the plugin host.
- **Refactor: desktop host boot logic moved out of generated strings into `host_boot`**: the boot behavior both desktop hosts (webview + CEF) used to carry as generated Python-in-strings (`SERVE_PY` / `PLUGIN_PY` / `DISPATCH_PY`) now lives in a real, tested module (`targets/desktop/native/host_boot.jac`). Generated hosts run a single statement (`_ctx = host_boot.boot('<config json>')`) and carry configuration only; any boot failure raises, deterministically gating the host's error path.
