- **Breaking: `region { }` blocks replaced by first-class `Region` handles and `in <handle> { }` opens**: Regions are now a complete feature (#7491). A `Region` is an ownable, sendable, escape-checked allocation extent opened with `in <handle> { ... }`; the old `region { ... }` soft keyword no longer parses (write `in Region() { ... }` for the anonymous form). On the native backend opens bump-allocate into a real arena reclaimed wholesale (LIFO dtor-log, one bulk free) at the handle's drop point; nodes and edges allocate into open regions, walkers grow them via `region_of(here)`, walkers themselves are now ARC-managed instead of immortal, `own <expr>` reboxes scalar/string copies out, and the Python backend fires `drop` hooks at portable points. The `E1307` escape rules are correspondingly stricter (opaque-call and aug-assign laundering, cross-membrane topology, and borrowed-handle sends are rejected).
