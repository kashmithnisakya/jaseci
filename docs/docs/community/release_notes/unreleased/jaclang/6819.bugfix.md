- **Fix: Windows native executable build now emits COFF instead of ELF (#6819)**: `jac nacompile`'s executable branch created its target machine with the default `codemodel="jitdefault"`. llvmlite appends `-elf` to the target triple when `os.name == "nt"` and the code model is `"jitdefault"`, so on a Windows host the executable build emitted an ELF object for a COFF target and the object-format guard tripped (the shared/DLL branch already passed `codemodel="small"` and was unaffected, but the exe is built first). The executable branch and `NaIRGenPass.postinit` now pass an explicit `codemodel="large"`. `"jitdefault"` resolves to the large model on x86_64, so this preserves the relocations the pure-Jac ELF linker already handles; the small/medium models emit `R_X86_64_32` absolutes it cannot resolve. No change on Linux: the native binary build is byte-for-byte identical.
