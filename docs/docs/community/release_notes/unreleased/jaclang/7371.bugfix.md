- **Fix: native pathway value semantics match CPython for comparisons, bool ops, membership, sort, and f-strings**: five silent-wrong-value families from jaseci-labs/jaseci#7320 Cluster 2. Chained comparisons (`a < b < c`) lowered only the first link, so `2 < 3 < 1` was True; every link is now lowered and ANDed with the middle operand evaluated once. `x or default` / `x and y` collapsed to a truth bit (ints came back 0/1, a returned `s or "fallback"` segfaulted); they now yield the deciding operand's value with short-circuit. `x in list[str]` and `k in d.keys()` compared pointer identity and were always False for heap strings; they now compare string values. `sort()`/`sorted()` silently ignored `key=` and `reverse=`; both are honored (stable in either direction), and unsupported keyword arguments to native methods are now a loud E5090. `range()` step, `str.find()` start, and `pow()` modulus were silently dropped; all three now work, including negative range steps. f-strings truncated at a fixed 1024-byte buffer and dropped format specs; they now size the buffer exactly and honor the common spec subset (`d`/`x`/`o`, `f`/`e`/`g` with precision, width/alignment, sign, zero-pad), with unsupported specs and `!r`/`!a` rejected at compile time instead of silently misrendering.
