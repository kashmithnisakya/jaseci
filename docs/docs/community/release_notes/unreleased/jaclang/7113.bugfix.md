- **Fix: stdlib serving robustness**: The zero-dependency stdlib server now binds a threading HTTP server (one SSE stream or slow handler no longer freezes every other request) and turns uncaught request errors into logged 500 JSON responses instead of silently dropped connections. A guest account whose persisted root anchor is missing from the anchor store (stale state from an older runtime) heals in place with a fresh root; a registered user in the same state gets a clear 500 naming the orphaned anchor instead of a raw `ValueError`. `sv import` microservices work without scale's extras: the stdlib server answers `/healthz`, core sv-to-sv RPC (including SSE re-streaming) runs on `http.client` instead of requiring httpx, and spawned sibling output goes to `.jac/logs/<module>.log` instead of being discarded. `/graph`, `/graph/data` and `/docs` are served by the stdlib server (previously scale-only). `zig build -Ddev` places bun into the linked source tree automatically, a missing bun is reported with one actionable message and retried per request (no server restart needed once fixed), and the startup "bundle already built" claim is now honest. The native GC-debug allocation-size probe keys on the target triple instead of the build host, so wasm modules no longer emit an unsatisfiable `env.malloc_size` import when built on macOS; the raylib_shooter web shim gained the `strlen`/`strchr`/`strtod`/`strtol` imports the na backend now emits.
