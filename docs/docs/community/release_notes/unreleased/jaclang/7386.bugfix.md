- **Fix: multi-file `jac fmt` parallelism actually engages on macOS/Linux**: the process-pool wrapper was built as a lambda that accepted no keyword arguments, so pool construction raised `TypeError` inside a broad `except` and every multi-file format run silently degraded to serial on any platform selecting a fork context; only Windows ever ran parallel. The pool is now constructed with `mp_context` passed directly (about 5x faster on an 8-core repo-wide `jac fmt`), the serial fallback reports its reason instead of swallowing it, and a mid-run pool failure can no longer double-format files through the serial retry.
- **Fix: Harden `jac fmt --cache` keying and edge cases**: Fix cache key collisions by incorporating effective `[format]`/`[check]` config (including suppress settings) and formatter-pipeline fingerprint. Under `--staged --verify`, cache on the staged blob digest (not the working-tree file) while preserving project logical paths. Annex failures are never cached as clean. Cache is skipped entirely with `--to_screen`.
