- **Bugfix: `jac install` installs the serve closure for jac-scale projects that resolve to `kind=cli`** (issue #7206): a jac-scale project with a plain `main.jac` and no explicit `[project] kind` inferred `kind=cli` (run action `execute`), and capability resolution only emits the `serve` capability for a serve action, so `jac install` skipped FastAPI/Uvicorn and the app crashed at serve time with `No module named 'fastapi'`. Capability resolution now treats any project with `[scale]`/`[plugins.scale]` intent as a serve action, so `jac install` closes over the serve deps (and `--scale` over the deploy clients) without changing how `jac run` executes such a project.
