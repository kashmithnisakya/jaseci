- **One artifact, one gate, a clean CLI (#7255)**: every project kind now builds through `jac build`, a fail-closed whole-program type-check gate, into a single self-describing `.jab` (the sealed image of #7142, packed as one deterministic file) that `jac run` / `jac start` execute or serve with **zero live compilation**: sealing bakes the prebuilt client dist, a serve manifest (client exports, access maps, endpoint effects, cross-boundary bindings), and per-triple AOT native binaries into the image, all hash-verified at load. Wheel / npm / self-contained binary / ejected source / native are projections of the same build via `jac build --as ...`. **Breaking: the CLI takes its clean break.** The surface drops to 24 visible intent-named core verbs. Renamed or folded: `format` is now `jac fmt`, `lint` is `jac check --lint`, `deps` is `jac install --plan`, `script` is `jac x <name>`, `jacpack` is `jac create --pack`, `enter` is `jac run --entry`, `debug` is `jac run --debug`, `bundle` and `eject` are `jac build --as ...`, and jac-scale's `destroy`/`status` are `jac scale destroy|status <app.jac>`; `js` is deleted. Removed verbs print their replacement and exit non-zero for this release, then disappear. `jac dev` is the new hot-reload dev loop; `nacompile`, `dot`, `browse`, `completions`, and `precommit` still work but are hidden from help. Plugins may register exactly one visible top-level noun, and CI now pins the verb budget.
