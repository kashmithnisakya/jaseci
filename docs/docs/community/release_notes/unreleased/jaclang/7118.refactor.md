- **Incremental JIR precompile**: the payload build's JIR precompile now skips every module whose existing `.jir` still passes its content-addressed module-key check (the same criterion the runtime uses to accept a bundled JIR), pruning orphans of deleted sources and stale python-version trees. A new `mkpayload --precompiled-cache=DIR` seeds and refreshes a persistent `.precompiled-build/` tree across builds, and CI caches it (with prefix-fallback restore keys, safe because entries are re-validated per module), turning the ~24-minute full precompile after a small jaclang change into a recompile of just the touched modules.
