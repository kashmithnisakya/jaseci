- **Breaking: Plugin/hook system removed; feature config flattened**: The pluggy-style plugin system has been removed entirely. `jaclang.jac0core.plugin`, `plugin_manager`, the `@hookable`/`@hookspec`/`@hookimpl` markers, entry-point plugin discovery, the `JAC_DISABLED_PLUGINS` env var, the `[plugins].disabled/enabled/discovery` keys, and the `jac plugins` command are all gone. The built-in features (byllm, scale, client/desktop, mcp, shadcn) are now called directly by core rather than dispatched through hooks, so external third-party plugins are no longer supported. Feature config sections have been flattened from the old `[plugins.<name>]` namespace to top-level `[<name>]` -- update `jac.toml` accordingly: the byllm section moves to `[byllm]`, the scale database section to `[scale.database]`, the client PWA section to `[client.pwa]`, and so on. Ejected SQL backends now inject their persistence via `JacRuntime.set_persistent_memory_provider(...)` instead of a plugin. See [Breaking Changes](../../breaking-changes.md).
