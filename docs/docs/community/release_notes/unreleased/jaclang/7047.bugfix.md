- **The `jac` binary no longer leaks `PYTHONHOME`/`PYTHONPATH` into child processes**: the single-binary launcher sets those two vars so its bundled CPython can find its own stdlib, but they were left in the process environment and inherited by every subprocess the app spawned. Any python-based child then adopted the jac runtime as its home and died with `Failed to import encodings module` (`ModuleNotFoundError: No module named 'encodings'`) -- this broke k8s exec-credential kubeconfigs (`aws eks get-token` / GKE `gcloud` / AKS `kubelogin`), surfacing as a `(401) Unauthorized` at database provisioning, as well as any other python tool a jac app shelled out to. The launcher now strips both vars from the environment immediately after `Py_Initialize` (once `sys.path` is frozen in memory), in both the CLI frontend and the desktop host, so children inherit a clean environment. Worker mode (`python`-drop-in re-spawns) is unaffected. This removes the need for the per-call-site `PYTHONHOME`/`PYTHONPATH` strips that jac-scale and the desktop dev launcher previously carried.
