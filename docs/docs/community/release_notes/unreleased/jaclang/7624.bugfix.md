- **Dynamic scheduler correctness**: cron weekday fields on `/jobs` now follow crontab semantics (numeric tokens are converted to day names before reaching APScheduler, so `0 0 * * 0` fires on Sunday, not Monday); one-shot `date` jobs transition to `completed` after firing instead of staying active forever and re-firing on restart; `POST/PUT /jobs` accept an optional `args` object passed to the walker/function on every run; and `update_job` raises like `create_job` when APScheduler is unavailable instead of updating the store without rescheduling.
