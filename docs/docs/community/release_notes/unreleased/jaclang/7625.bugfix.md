- **Scheduled jobs run on exactly one replica**: a Mongo-backed leader lease (`scheduler:<module>`, TTL via `[scale.scheduler] leader_ttl`, opt out with `coordination = "none"`) gates static registration, dynamic restore, and the APScheduler; followers stay idle, reconcile leader-side picks up jobs created via any replica, and a killed leader fails over within one lease TTL. sv sibling processes no longer run a second scheduler (`JAC_SV_SIBLING=1` guard), get process-group cleanup plus a parent-death watchdog instead of leak-prone atexit-only termination, and a Mongo connection failure now logs an error (or fails startup with `require_persistence = true`) instead of silently downgrading to the in-memory job store.
