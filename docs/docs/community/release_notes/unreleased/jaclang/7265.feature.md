- **WebSocket hardening**: production-ready WebSocket auth and limits. A JWT handshake (query-token with first-frame fallback, tagged reject reasons, and a `refresh` control message), per-target / per-user / per-anonymous connection caps, message-size, rate, and per-send limits, heartbeat with idle close, a pluggable in-process or Redis pub/sub broadcast backplane selected via `[scale.websocket]`, and Prometheus WebSocket metrics. Tokens are validated once at the handshake rather than on every frame, and a session that outlives its token is closed with `4401` within one heartbeat interval; send a `refresh` frame before expiry to stay connected. A backplane that cannot be subscribed now rejects the connection with `SERVICE_UNAVAILABLE` / `4503` instead of masquerading as a capacity limit. See the [WebSocket guide](https://github.com/jaseci-labs/jaseci/blob/main/jac/jaclang/scale/docs/websocket-guide.md) for the full frame protocol, error codes, and close codes.
