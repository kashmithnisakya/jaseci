- **WebSocket hardening**: production-ready WebSocket auth and limits. A JWT handshake (query-token with first-frame fallback, tagged reject reasons, and a `refresh` control message), per-target / per-user / per-anonymous connection caps, message-size, rate, and per-send limits, heartbeat with idle close, a pluggable in-process or Redis pub/sub broadcast backplane selected via `[plugins.scale.websocket]`, and Prometheus WebSocket metrics.
