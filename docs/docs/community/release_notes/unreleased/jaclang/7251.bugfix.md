- **`jac eject` no longer ships test code or `.jac` source paths in the backend**: The compiler merges `.impl.jac` / `.test.jac` annexes into their parent module, so ejected modules previously carried `@jac_test` functions and `@impl_patch_filename('.../foo.impl.jac')` decorators. Those were absolute paths in the project's own modules (leaking the author's filesystem layout) and relative `jaclang/...` paths across the vendored runtime. Every emitted app module and every vendored jaclang runtime module is now sanitized: test functions are dropped, the dev-path decorators are removed (tracebacks resolve to the emitted `.py`), and the unused `jac_test` / `impl_patch_filename` names are pruned from `jaclang` imports (a project's own symbol of the same name is left untouched).
