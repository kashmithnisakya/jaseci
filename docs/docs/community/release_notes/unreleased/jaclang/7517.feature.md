- **Production-grade `jac build --as source`**: conversion now runs a capability audit that refuses (with a per-module report) any feature the source projection cannot lower, instead of silently demoting it; `[eject] allow_unsupported = true` converts with those symbols omitted. The generated FastAPI app now has behavioral parity with `jac start`: scale-compatible transport envelopes, `@schedule` jobs wired to the core scheduler, `protocol=WEBSOCKET` walkers as real `/ws/` routes (token handshake, broadcast fan-out), `protocol=WEBHOOK` walkers behind X-API-Key + HMAC-SHA256 + replay verification with `/api-key/create` included, `@restspec` walkers mounted at their declared method/path, multipart signatures for `UploadFile` walker fields, and idiomatic adapters for `kvstore` (redis-py) and `publish`/`@subscribe` (in-process broker).
- **Meaning-typed fidelity for converted apps**: the compiler-captured MTIR map is emitted as readable `backend/mtir_data.py` and hydrates a new runtime `MTIR_OVERLAY` at boot, so `by llm()` prompts and schemas carry the same semstrings and parameter/return-type structure as under `jac start`, including semstrings on nested fields and enum members. It is emitted as plain source rather than a pickle so the metadata stays reviewable and diffable, and the converted app never unpickles a binary at startup.
- **Fix**: a backticked `` `root `` entry trigger compiled to the jaclib `root()` function instead of the `Root` class, failing at spawn with an `issubclass()` TypeError.
- **Fix**: conversion read the runtime MTIR map destructively, so a second `eject()` in the same process, and any `by llm()` call after conversion began, silently lost their prompt metadata and fell back to bare introspection.
- **Fix**: `.na.jac` native modules were skipped entirely, but server modules importing them still emitted the import, so the converted app died at boot with `ModuleNotFoundError`. A single `server` flag was gating both "write this module" and "expose its symbols as endpoints"; those are now separate. Native modules are emitted as ordinary Python (their lowering is already complete, it just runs interpreted rather than AOT-compiled) while staying out of the API surface.
